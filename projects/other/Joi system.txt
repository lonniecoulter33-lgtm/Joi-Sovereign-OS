YOUR CURRENT MODEL:
  mistral-7b-instruct-v0.1.Q4_K_S
  └── v0.1 = first release, weakest instruction following
  └── Q4_K_S = 4-bit small = ~3.7GB, fastest, lowest quality

DIRECT UPGRADES (same download size, better performance):
  mistral-7b-instruct-v0.3.Q4_K_M  ← v0.3 is much better at following
                                       instructions. Q4_K_M is ~4.2GB,
                                       slightly bigger but noticeably smarter.
                                       BEST EASY UPGRADE.

BETTER MODELS (if your PC can handle ~4-5GB models):
  llama-3.1-8b-instruct.Q4_K_M     ← Meta's model. Better at tool calling
                                       than Mistral. Better at writing too.
                                       This is what I'd actually recommend.

  qwen2.5-7b-instruct.Q4_K_M       ← Strong at structured output and
                                       following complex instructions.
                                       Good alternative.

  phi-3.5-mini-instruct.Q4_K_M     ← Only 3.8B parameters but surprisingly
                                       capable. Smallest, fastest option
                                       if your PC is slower.

RULE OF THUMB FOR QUANTIZATION:
  Q4_K_S = smallest, fastest, weakest    (what you have now)
  Q4_K_M = slightly bigger, noticeably better  ← sweet spot
  Q5_K_M = bigger still, best quality at ~5GB
  Q8_0   = near-perfect quality, ~8GB, needs good RAM


1. BETTER MODEL FILES (what we just covered)
   └── Upgrade from v0.1 to v0.3, or switch to Llama 3.1
       This is the single biggest improvement. Do this first.

2. SYSTEM PROMPT ENGINEERING  
   └── We write a really strong, detailed system prompt specifically
       for Mistral. Different models respond to instructions differently.
       A prompt tuned for Mistral makes it perform 30-40% better at tasks.
       We build this into joi_llm.py.

3. FEW-SHOT EXAMPLES
   └── When Joi sends a request to Mistral, she includes 2-3 examples
       of "here's what a good response looks like." This teaches the model
       the quality bar without any training. We can do this automatically.

4. OLLAMA (alternative to LM Studio)
   └── Free, open source, runs models locally same as LM Studio.
       Slightly more control. You can run multiple models simultaneously.
       Worth looking into later but LM Studio is fine for now.

5. LONG-TERM: FINE-TUNING
   └── If you ever get a machine with a decent GPU (even a 3060/4060),
       you can fine-tune a model on your own writing style using tools
       like "llama.cpp" or "Hugging Face." That literally teaches the
       model to write like YOU. That's the endgame for a truly personal AI.
       Not now, but it's the roadmap.

1. Get your Gemini API key (free, takes 2 minutes at aistudio.google.com)

2. We build joi_llm.py — the central router that:
     • Tries Mistral first for everything
     • Routes writing to Claude in safe chunks
     • Routes research/summarization to Gemini
     • Falls back to GPT-4o-mini (not 4o) for tool decisions
     • Only uses full GPT-4o for DALL-E
     • Tracks which AI worked best for what, and learns from it

3. We update .env with:
     • LM_STUDIO_URL + model name
     • GEMINI_API_KEY
     • Switch default model from gpt-4o to gpt-4o-mini

4. The modular split + Phase 2 app launcher on top of that
